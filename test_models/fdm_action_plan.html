Step by step action plan for FDM Checker

Click each phase to expand. Do them in order.

1
Deploy live — get a real URL today
~30 minutes · free · do this first
▶
1
Do now
Push your code to GitHub
Create a free GitHub account if needed. Create a new repo called fdm-checker, make it public.
git init git add . git commit -m "initial commit" git remote add origin https://github.com/YOURNAME/fdm-checker.git git push -u origin main
2
Do now
Deploy frontend to Vercel (free)
Go to vercel.com → New Project → import your GitHub repo. It auto-detects Vite/React. Hit Deploy. You get a live URL in 2 minutes.
✓ Result: yourapp.vercel.app — shareable URL you can show anyone
3
Do now
Deploy your Python backend to Render (free)
Your app has a FastAPI/Flask backend at /api. Go to render.com → New Web Service → connect GitHub → set start command. Free tier sleeps after inactivity but wakes in ~30s — fine for testing.
Start command (FastAPI): uvicorn main:app --host 0.0.0.0 --port $PORT Start command (Flask): gunicorn app:app
Then in Vercel, add an environment variable: VITE_API_URL=https://your-backend.onrender.com
2
Make it actually useful — 3 day code sprint
3 days · the difference between a toy and a tool
▶
1
Day 1
Add printer profile selector to the UI
Add a dropdown above the drop zone. Options: Bambu Lab FDM / Prusa FDM / Formlabs Resin / Generic FDM. Store as state, pass to results. This makes every result feel personalized. Tell Claude: "Add a printer profile selector dropdown to my App.js — here is the code [paste it]. Store the selected profile in state and pass it to SingleMode and BatchMode as a prop."
✓ Now users feel seen. "Oh it knows I have a Bambu."
2
Day 1
Replace jargon with plain English + fix instructions
Every IssueRow currently shows raw text like "non-manifold edges detected." Change it to: "Mesh has holes — your slicer will behave unpredictably. Fix: open in Meshmixer → Analysis → Inspector → Auto Repair All." Ask Claude to write a humanizeIssue(text, profile) function that maps issue text to plain descriptions + fix steps.
✓ Now a dental tech or hobbyist actually knows what to do
3
Day 2
Add a printability score (0–100)
Compute a single score from wall thickness, overhang %, and watertight status. Show it as a big number with a color ring. People screenshot and share scores. "My model scored 91!" is social. Ask Claude: "Add a printability score 0-100 to my results display. Weight: watertight 40pts, wall thickness 35pts, overhang 25pts. Show as a large colored circle."
✓ Makes results shareable. Drives word-of-mouth.
4
Day 3
Add material cost estimate
You already have volume_cm3 from the API. PLA density = 1.24 g/cm³, resin = 1.1 g/cm³. PLA costs ~$0.02/g, resin ~$0.13/g. Show: "Estimated material: 18g (~$0.36 PLA) — a failed print wastes this + your time." This is the ROI argument that justifies paying.
✓ Users now understand what a failed print actually costs them
3
Get your first 100 users — free marketing
1 week · no budget needed
▶
1
Post day 1
Post on r/3Dprinting (1.8M members)
Title: "I built a free tool that checks if your STL will actually print before you waste filament — roast it". Include a GIF of the app working. The word "roast" gets engagement, the word "free" gets clicks. DO NOT mention pricing yet.
✓ Expect 200–2000 visitors in 48 hours if it resonates
2
Same day
Post on r/FDMprinting, r/prusa3d, r/BambuLab
Each subreddit has a different angle. For r/BambuLab: "I added Bambu Lab printer profiles so thresholds match your machine." Community-specific framing gets 3x the upvotes.
3
Day 2
Post on Printables.com and Thangs.com
These are model-sharing platforms. Users are downloading and printing daily — they are your exact target. Both have community forums. Post in the "tools" or "makers" section.
4
Day 3
Add a simple email capture
Use Mailchimp free tier (0–500 emails free). After analysis completes, show: "Get notified when we add auto-repair and PDF reports — enter your email." This is your list. When you launch paid features, you email this list first.
✓ 50 emails = real validation. 200 emails = launch paid tier.
5
Day 5
Cold email 10 dental labs
Google "dental lab [your city]". Email the owner/manager. Subject: "Free printability check for your STL files — no strings". Body: 3 sentences max. Offer to check 5 of their files free. One yes = a case study. One paying lab = $99–199/month.
✓ 1 paying dental lab covers your server costs immediately
